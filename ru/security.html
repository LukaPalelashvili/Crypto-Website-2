Благодаря браузеру Web 3.0, PCNP обеспечивает доступ к онлайн-информации для своего пользователя без вмешательства третьих лиц в ваши личные данные. Нет недостатка или разницы в стандартах безопасности между кошельком PCNP и браузером. 

Безопасная зашифрованная идентификация PCN 

Пользователи PCNP получают криптографическую пару ключей при создании учетной записи, которая защищена паролем. Пользователи могут добавлять всю информацию в свой профиль PCNP. Профиль конечного пользователя может быть публичным или приватным, как вы хотите. 

Протокол связи P2P

Для передачи целевых сообщений PCNP использует протокол Woku, который зашифрован и открыт только для предполагаемых получателей. Текстовая информация пользователей PCNP остается в безопасности благодаря устранению сторонних точек доступа. Почтовые серверы Woku не полностью основаны на одноранговом протоколе, так как он сохраняет ваше сообщение, когда получатель находится в автономном режиме, и пересылает его, когда он находится в режиме онлайн.

Протокол безопасной передачи данных от конца к концу

End-to-End шифрование генерирует открытый ключ, разделяемый между отправителем и получателем, поскольку это позволяет сообщению быть эксклюзивным для обоих и расшифровываться только контактом получателя.

Дополнительная безопасность с PFS

Совершенная передовая секретность обеспечивает безопасность ваших прошлых и будущих данных даже в случае компрометации сеансового ключа. PFS служит дополнительным гарантированным уровнем безопасности с особым согласованием ключей для обмена сообщениями PCNP один на один.

Псевдо-анонимные учетные записи PCNP

PCNP не будет запрашивать вашу личную информацию для создания нового аккаунта. Для регистрации нам не нужны никакие сторонние подтверждения, например, ваш контактный номер или электронная почта. У пользователя есть только ключи и мнемонические фразы для получения доступа к учетной записи PCNP. 

У пользователей нет двухфакторной аутентификации или системы восстановления пароля на случай, если вы забудете какой-либо из своих ключей или фраз. Пользователям рекомендуется хранить пароли, ключи и фразы в автономном расширении на случай их забывания. 

Защита ключа пользователя PCNP 

Открытые и закрытые ключи пользователей PCNP хранятся в файле keystore json локально в памяти устройства. Мы никогда не будем использовать сторонние сервисы для хранения и управления открытыми и закрытыми ключами пользователей.  После создания учетной записи PCNP первый ключ BIP44 сохраняется на устройстве пользователя локально.

Файл json зашифрован вашим паролем и доступен пользователю только через приложение PCNP. Наша система всегда будет отдавать приоритет хранению особо важных данных в первую очередь на вашем устройстве. KeyCard - это дополнительное аппаратное расширение, добавляющее еще один уровень безопасности для бесконтактных платежей, управления и хранения закрытых ключей. 

Для получения дополнительной информации о Keycard посетите keycard.tech.

Безопасный инкогнито веб-браузинг 3.0 

PCNP безопасный браузер Web 3.0 с включенным по умолчанию режимом конфиденциальности для конечных пользователей. Браузер PCNP держит пользователей в курсе, а их кошельки защищены, поэтому DApps разрешают соединение с кошельком PCNP, и DApp может сломаться, если не совместим. 

Для экономии платы за газ и сокращения количества транзакций на блокчейне браузер PCNP адаптировал и использует EIP712. Браузер PCNP стремится повысить удобство использования с помощью EIP417, используя подписание сообщений вне цепочки для использования на цепочке. В настоящее время сообщения PCNP подписываются непрозрачной шестнадцатеричной строкой, отображаемой пользователю с минимальным контекстом о том, из чего состоит сообщение.

Защита с помощью кошелька PCNP, не подлежащего опеке 

Пользователи кошельков PCNP имеют абсолютную власть над своими средствами и транзакциями без использования какого-либо сервера. Только с помощью мнемонической фразы пользователя и закрытых ключей криптокошелек доступен, так как данные шифруются на вашем устройстве. 

Пользователям рекомендуется дополнительно сохранять закрытый ключ и фразу в автономном хранилище на случай, если пользователь кошелька PCNP потеряет или забудет эти данные. 

Предотвращайте фишинговые атаки с помощью фразы из 3 слов для входа в систему

Для подписания всех криптотранзакций пользователи PCNP должны ставить подпись, используя случайно сгенерированную трехсловную фразу. Антифишинговая трехсловная фраза сохраняется локально на устройстве каждый раз, когда пользователь инициирует транзакцию.

Если в случае, если пользователь не распознает фразу из 3 слов или ему не предъявляется никакой фразы. Пользователям PCNP рекомендуется немедленно отменить транзакции, выйти из аккаунта и сообщить нам.
